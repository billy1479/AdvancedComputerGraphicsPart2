Here's how the implementation draws on the lecture material:

**The Visualisation Pipeline (Lecture 2)** is the backbone of the whole program. The code follows the exact sequence taught: data acquisition (rasterio reads raw LROC TIFFs and extracts geospatial metadata), data filtering (NaN replacement for sentinel values, subsampling for contour performance), data mapping (scalar elevation values mapped to colours and contour lines), and rendering (matplotlib produces the final 2D image). The `DataStore` and `LunarDataset` classes directly mirror the pipeline's modular decomposition — each stage is a separate responsibility.

**Colour Mapping (Lectures 3–4)** is applied through the colourmap selector. The lectures warn that rainbow colourmaps have poor perceptual ordering and non-uniform luminance, so the implementation defaults to `terrain` and offers perceptually uniform alternatives like `viridis` and `cividis`. It also includes diverging scales (`coolwarm`, `RdYlBu_r`) which the lectures describe as useful for emphasising deviation from a middle value — relevant here for distinguishing highs and lows relative to a mean elevation. The greyscale option connects to the lecture point that luminance-based maps have natural ordering, even if they sacrifice dynamic range.

**Contouring / Isolines (Lecture 4)** is implemented directly. The lectures define contours as the set of all points where f(x) equals a given isovalue, and describe the marching squares algorithm for computing them on quad grids. The code uses matplotlib's `contour()` which implements this internally. The adjustable interval slider lets users control which elevation values generate contour lines — exactly the parameter the brief asks for. The lecture's observation that colour banding and contouring are related is exploited by overlaying contours on top of a smooth colourmap, giving both overview (colour) and precise value tracing (contour lines).

**Height Plots and Displacement Maps (Lecture 4)** inform the design even though the 3D perspective view is Part B. The current 2D colour-mapped view is the "flat" counterpart — the lectures present these as complementary, noting that 2D colour maps preserve spatial accuracy while 3D displacement plots add shading cues for fine-grained detail at the cost of occlusion and distortion.

**Inverse Mapping (Lecture 2)** is what the click-to-sample feature implements. The lectures describe visualisation as a function v: D^m → R^5 (data to image), and analysis as the inverse v^{-1}: D^5 → R^m (image back to data). When the user clicks on a coloured pixel and the code inverts the affine transform to retrieve the underlying elevation and illumination values, that's exactly this inverse mapping in action.

**Multi-scale / Overview-to-Detail (Lectures 1–2)** is addressed by the three spatial scale levels and the zoom mechanism. The lectures emphasise that effective visualisation presents information at several levels of detail, from overviews to fine structure. The radio buttons switch between regional, intermediate, and detailed datasets, while the right-drag zoom lets users drill further into sub-regions within any scale — implementing semantic zoom where contour density and detail adapt to the viewed extent.

**Multi-variate Visualisation (Lectures 3–4)** is applied through the illumination overlay. The lectures discuss the challenge of displaying multiple scalar fields on the same domain. The implementation uses transparency blending — elevation is the base layer with one colourmap, illumination is overlaid semi-transparently with a separate colourmap and adjustable opacity. This lets users visually correlate terrain shape with solar illumination, which is the core multi-variate question for Artemis III site selection.

**Effective Colourmap Design Goals (Lecture 3)** shaped specific choices. The lectures list five goals: telling absolute values, value ordering, value differences, identifying selected values, and showing rate of change. The terrain colourmap handles ordering well through luminance progression. Contour lines address the "selected values" goal by marking exact elevation levels. The illumination overlay with a sequential `hot` colourmap prioritises ordering (dark = shadowed, bright = illuminated) — matching the lecture advice to use luminance for clear ordering in compact domains.
